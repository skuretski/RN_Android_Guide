Getting Started

First Steps

Getting set up requires a few steps, regardless of iOS or Android. Naturally, the Android set up has some distinct differences. Use this guide from React Native to get set up. The official docs let you pick your target mobile OS (Android or iOS) and your development OS (Mac, Linux, or Windows). For our purposes, select Android as your mobile OS and your corresponding development OS. Follow the instructions laid out from React Native. Let's discuss some of the set up steps!

Node.js

Mac and Homebrew

If you are developing on a Mac, you can use Homebrew to install Node.js if you don't have it already. If you don't want to use Homebrew, you can also follow the Node.js instructions on how to download and install without it.


Windows and Chocolatey

For Windows installation of Node.js and other packages, you can use Chocolately. You can also follow the Node.js instructions on their site to download and install.

Watchman

Watchman is helpful for development since it will detect changes in your files. It's useful for debugging, hot reloading, and live reloading. It should be noted Watchman is supported on Mac and Linux. Windows support is currently in Alpha. Watchman will (hopefully) speed your development up by detecting these changes. If you use a Mac or Linux, it is highly suggested.


React Native CLI

Don't forget to install the React Native command line interface! Otherwise, you won't be able to run your project. Install using your Node.js command prompt with npm install -g react-native-cli. This will install globally (from the -g flag), but this is what we want!


Text Editors and Android Studio

You can use any text editor you want, like Sublime Text, Notepad++, Android Studio, Emacs, Vim, etc. I personally use Sublime Text 3 and installed some packages to help with syntax. Use or not use packages; it's your preference! I found Babel to be lightweight and useful. It helps with ES6 JavaScript and JSX syntax. **Note: Babel requires Sublime Text 3. Under Sublime's "Preferences" tab, go to "Package Control" and select "Install Package". Search for Babel. You might find linters helpful as well, like ESLint for ES6 and SublimeLinter.


Even if not developing in Android Studio, it is very useful to download and set up since you can manage your Android SDK, APIs, and emulators. It is very important to set up your path to your Android SDK, otherwise you will not be able to run the app.


Running Your First App

Make sure to have your emulator or device running before running your application!

If you did the react-native init <YourProjectName> in your Node.js console window, cd <YourProjectName> into that directory, and then entered react-native run-android, running an application for the first time will take a bit of time (about 1-5 min). The length of time will depend on your processor and having an accelerator. I have an Intel i7-4770K and enabled Intel HAXM. Debugging

Debugging in React Native is easy to do. This documentation page goes over the features. You might think, "Oh myyy, getting the application to run took forever! Debugging will take so long!" Luckily, with Hot Reloading and Live Reloading you won't have to restart it each time you make a change. **Note: you cannot make changes to the store and expect reloading to work. You might get the Red Screen of Death.


For Android, hitting ctrl + m on your keyboard will open the Developer Menu and you can select the different debugging options. Remote JS Debugging will open a Chrome tab where you can debug remotely and see your console.log("foo bar").


The Ever Evolving React Native

Be sure to check GitHub or StackOverflow for certain features if you're having issues because Facebook is constantly revising, updating, and changing React Native. These revisions and additions may have some bugs since React Native spans across iOS, Android, Java, JavaScript, ES6 changes, and so on. For example, the Geolocation API for Android has known issues with timing out with Android API 21, 22, and 23. Issue on GitHub here. Since React Native is open source, you can view known issues under "Issues" in GitHub. At time of writing (Nov 2016), there are 683 listed open issues and 5960 closed issues.
